# Guia de Contribuição

Obrigado por considerar contribuir com nossa plataforma! Este documento fornece as diretrizes para contribuir com o projeto.

## Como Contribuir

1. Fork o projeto
2. Crie uma branch para sua feature (`git checkout -b feature/AmazingFeature`)
3. Commit suas mudanças (`git commit -m 'Add some AmazingFeature'`)
4. Push para a branch (`git push origin feature/AmazingFeature`)
5. Abra um Pull Request

## Padrões de Código

- Use TypeScript
- Siga o estilo de código existente
- Mantenha os commits atômicos e bem descritos
- Adicione testes para novas funcionalidades

## Reportando Bugs

1. Verifique se o bug já não foi reportado
2. Inclua passos detalhados para reproduzir
3. Inclua logs de erro se disponíveis
4. Descreva o comportamento esperado

## Sugerindo Melhorias

1. Descreva o problema que sua melhoria resolve
2. Explique como sua sugestão beneficia o projeto
3. Seja específico e forneça exemplos

## Processo de Review

1. Dois aprovadores são necessários
2. Todos os testes devem passar
3. O código deve seguir os padrões do projeto
4. A documentação deve ser atualizada

## Dúvidas?

Entre em contato com os mantenedores do projeto.
